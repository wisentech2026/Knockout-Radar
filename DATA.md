# 数据与模型说明

## 数据口径

| 数据 | 来源 | 更新 | 真实性 |
|---|---|---|---|
| 分组、赛程、对阵树 | openfootball `worldcup.json`（公有领域） | 每日 | 真实 |
| 已赛比分 | 同上，开赛后逐场写入 | 每日 | 真实 |
| 球队强度（Elo）与天气修正 | `data/teams.json` 手工快照 | 手动 | 估计值 |
| 全部概率 | 蒙特卡洛模拟统计 | 每次重跑 | 模型输出 |

## 模型（V4）

1. **小组赛**：未赛场次用泊松进球模型，期望进球 `λ = 1.3 × e^(ΔElo/500)`；缩放常数 500 经数值标定，使模型隐含胜率与 Elo 期望在 0–400 分差内对齐于 2–3 个百分点。比分分布带 Dixon-Coles 低比分修正（ρ = −0.1），平局率落在世界杯历史区间。已赛场次直接采用真实比分。
2. **赛中 Elo 动态更新**：每场真实赛果按日期回放，用世界杯口径（K=60，净胜球倍率）更新 Elo；模拟、撞车判定、强强对位全部基于更新后的 liveElo。小组赛打得越多，预测越贴近当届真实状态。
3. **主场按场地逐场判定**：不再一刀切。16 个场地映射到美/墨/加三国，东道主只在本国场地获得 +50；墨西哥在墨西哥城（海拔 2240m）额外 +25。淘汰赛场地随对阵树固定，因此「墨西哥若守住蒙特雷半区则保留主场」这类路径效应会被如实模拟。
4. **出线**：12 组前二 + 8 个最佳第三名（积分 → 净胜球 → 进球）；第三名落位按赛程文件中每个槽位的分组约束回溯匹配。
5. **淘汰赛**：按真实对阵树推进；打平进入点球，胜率向 50% 压缩 40%。
6. **场景开关**（四套预生成结果）：天气适应（±15 Elo，`teams.json` 的 `weatherAdj`）× 主场优势。
7. **派生指标**：撞车风险 = 32/16 强遇 Elo 前八的概率；路径判断 = 16 强概率前 16 的球队中，32 强对手池平均 Elo 的四分位比较；爆冷可能 = 淘汰赛前两轮击败 Elo 更高对手的频率。

## 数据缺口清单

| 缺失数据 | 对精度的影响 | 现状与补法 |
|---|---|---|
| 自动化 Elo 源 | 高：开赛前的概率完全依赖快照质量 | eloratings.net 无 API；`teams.json` 手工校准（48 个数字约五分钟）。开赛后影响递减——liveElo 会被真实赛果接管 |
| 伤停与阵容 | 中：核心球员缺阵约值 30–80 Elo | 未接入。可在 `teams.json` 加临时修正项即刻生效；自动化需付费数据源 |
| 博彩市场赔率 | 中：市场是最强的单一预测源 | 未接入（需 API Key）。可按 50/50 与 Elo 期望加权混合，只作输入特征 |
| 逐场天气预报 | 低：当前用球队级适应性近似 | 场地已知，可接天气 API 做场级修正 |
| 比赛日间隔/行程 | 低 | 赛程数据里有日期与场地，可计算休息天数差，目前未启用 |
| 红黄牌停赛 | 低 | 需逐场事件数据，未接入 |

## Elo 快照怎么更新

`data/teams.json` 是唯一需要人工维护的文件。建议小组赛每轮结束后参考 eloratings.net 校一次（48 个数字，约五分钟）。开赛后真实比分占比上升，Elo 误差的影响会自然衰减。

## 升级路线（可选）

- **更高频比分**：API-Football 免费档每天 100 次请求、15 秒级更新；用 Cloudflare Worker 存 Key 并缓存，替换 `fetch-fixtures.mjs` 即可，页面无需改动。
- **赔率输入**：可把市场隐含概率混入单场胜率，仅作为模型输入特征；页面定位是赛果概率展示，不输出投注建议。
- **伤停与状态**：在 `teams.json` 加临时 Elo 修正即可生效。

# 2026 World Cup Knockout Radar

2026 世界杯淘汰赛路线预测。纯静态网页，可直接部署到 GitHub Pages、Cloudflare Pages、Vercel 或任意静态托管。

## V6 版本说明

**视觉重构**（基于 2026 设计趋势：字体即界面、无图片资产、轻量 CSS 逻辑）：
- 严格三色，全部来自世界杯语义：草皮绿（晋级/主交互）、信号红（撞车/风险）、奖杯金（仅品牌标与冠军两处）。
- 全屏排版式头部，无头图；球场中圈线稿由纯 CSS 生成。
- 单流布局 + 吸顶导航（滚动定位高亮），替代横幅 bento + 底部 dock 双导航。
- 流式断点 1024 / 760 / 560 / 400，单一 DOM 自适应，热力图行列头双向吸附。
- 热力图点选即明示：「经过 20,000 次模拟，两队相遇 N 次——概率 X%」，行列联动高亮。
- 强强预警卡含 90 分钟胜平负三分条、最可能比分、相遇次数。

**多语言**：中文 / English / Español / 粵語 四语完整支持（粤语为地道粤文书写 + 香港足球媒体术语：對碰、十二碼、打和、籤運、準決賽、蒙地卡羅，队名按港译：象牙海岸、沙地阿拉伯、南韓、澳洲等），右上角切换，默认跟随浏览器语言并记忆选择。引擎输出语言无关的结构化数据（48 队三语队名在 `data/teams.json`），全部文案由前端 `app.js` 顶部的字典渲染——增改语言只动这一个文件。Buy Me a Coffee 链接在 `app.js` 顶部 `BMC_URL` 一处配置。

**数据与模型**：

- 概率不再手填：48 队 × 104 场真实赛程（openfootball 数据集，含 FIFA 实际抽签的分组与对阵树）跑两万次 Elo 蒙特卡洛模拟，晋级榜、相遇热力图、强强预警、路径与爆冷全部出自同一组模拟，彼此自洽。
- 开赛后自动计入真实比分，只模拟剩余赛程；附带 GitHub Action 每日两次自动更新。
- 页面上不再有任何写死的球队名或概率（V1 中"法国 vs 德国"等静态标题已全部动态化）。
- 两个开关现在是真实模型参数：天气适应（±15 Elo）与主场优势（+50 Elo），对应四套预生成场景。
- 适配修复：热力图行头横滚吸附、iOS 安全区、补充中文字体回落栈。均不改变视觉样式。

## 文件结构

```text
.
├── index.html / styles.css / app.js   页面（视觉 = V1，数据读 data.json）
├── data.json                          页面唯一数据源（生成产物）
├── assets/                            V1 原版视觉资源
├── data/
│   ├── fixtures.json                  openfootball 赛程快照
│   └── teams.json                     48 队 Elo + 天气修正（手工维护）
├── scripts/
│   ├── fetch-fixtures.mjs             拉取最新赛程与比分
│   └── simulate.mjs                   蒙特卡洛引擎
└── .github/workflows/update-data.yml  每日自动更新
```

## 本地预览

```bash
npm run serve      # http://127.0.0.1:8766
npm run update     # 拉取最新赛果 + 重跑模拟（Node 18+）
```

## GitHub Pages 部署

1. 推送本目录到仓库 `main` 分支。
2. `Settings → Pages → Deploy from a branch → main / root`。
3. 仓库自带的 Action 会在世界杯期间每天两次拉取真实比分、重跑模拟并提交新的 `data.json`，Pages 自动重新发布。零服务器、零 API Key。

## 重要说明

页面展示的是概率模型的统计结果，不是官方预测，不构成任何投注建议。
